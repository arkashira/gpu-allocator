## Test Plan — Permission Management System (`gpu-allocator`)

### 1. Acceptance Criteria

- A user record can be created, retrieved, updated, and deleted via the permission management interface with changes reflected immediately in the data store
- Permission grants are enforced at access time — a user without a required permission receives a 403/denied response, not a degraded success
- Every mutating action (create, update, delete, grant, revoke) produces a timestamped audit log entry containing actor, target, action, and outcome
- Permission checks are role-aware: a user with an admin role inherits downstream permissions without explicit grants
- Concurrent permission revocations take effect within one request cycle — no stale-cache window allows a revoked user to proceed
- All user-facing input is validated and rejected with a descriptive error before reaching the data layer (no raw exceptions propagate)
- Audit logs are append-only and cannot be deleted through the same interface that writes them
