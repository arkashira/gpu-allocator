### 1. Acceptance Criteria
1. **Pattern Detection** – The detector flags a GPU allocation session as suspicious when the usage pattern deviates from the baseline by > 30 % in any of the following dimensions: average utilization, allocation duration, or geographic IP mismatch, **within a 5‑minute sliding window**.  
2. **Alert Generation** – Upon detection, an alert object is created and persisted to the `alerts` table with severity “high”, a timestamp, and a reference to the offending allocation ID.  
3. **Real‑time Delivery** – The alert is published to the `notifications/diversion` channel **within 2 seconds** of detection.  
4. **Idempotency** – The same allocation session must not generate duplicate alerts; once an alert exists for a given session, further detections are ignored.  
5. **Configurable Thresholds** – Threshold percentages and time‑window size are configurable via `config.yaml` and default to the values in (1).  
6. **Observability** – Metrics `diversion.detected`, `diversion.alert_sent`, and `diversion.false_positive` are emitted to Prometheus with appropriate labels.  
7. **Security** – Alert payloads are signed with the system HMAC key; verification must succeed for any consumer.
